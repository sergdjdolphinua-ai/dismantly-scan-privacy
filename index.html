Privacy Policy

Dismantly Scan — Chrome Extension · Last updated: May 14, 2026

This Privacy Policy describes what data the Dismantly Scan Chrome Extension (the “Extension”) collects, how it is used, where it is stored, and the choices you have. The Extension is published by Serhii Hetmanskyi (“we”, “us”). For any questions, contact: sergdjdolphinua@gmail.com.

1. What the Extension does

The Extension is a productivity tool for warehouse workers using app.dismantly.ai. It reads QR codes and barcodes, opens the matching part page, and automates user-selected workflows (View, Add Images, Work Order, Delete). It only operates while the user is interacting with app.dismantly.ai.

2. Data we collect

The Extension collects the following categories of data, exclusively to provide the features the user has enabled:

  • Personally identifiable information — the user's name and email address as exposed by the user's own app.dismantly.ai session (Amazon Cognito user attributes). Used to attribute photo sessions to the correct photographer in the in-extension dashboard.
  • Authentication information — the JWT token of the user's active app.dismantly.ai session is copied from the page's localStorage into chrome.storage.local. It is used solely to authenticate API calls the Extension makes to api.dismantly.ai on the user's behalf.
  • User activity — a log of photo sessions (photoLog): part ID, number of photos uploaded/deleted, timestamps, and bonus calculations. Used for the local productivity dashboard and optional payroll reporting.
  • Website content (metadata) — values read from app.dismantly.ai pages: part_id, part tag, Reference number, first photo URL, sold-out and deleted flags. Used to fill in the local log and to power Work Order printing.

The Extension does not collect health data, financial or payment information, personal communications, location, browsing history outside of app.dismantly.ai, or any data from other websites.

3. Where data is stored

  • Locally on your device — preferences, photoLog, and the auth token are kept in Chrome's chrome.storage.local and chrome.storage.sync. They never leave the device unless an optional sync feature (below) is explicitly enabled.
  • api.dismantly.ai — when the user triggers a sync, the Extension calls the user's own Dismantly account API to refresh part metadata. This is the same backend the user is already authenticated with by using app.dismantly.ai.
  • Google Sheets (optional, admin-configured) — if an administrator configures the Google Sheets sync, photoLog records are sent via a Google Apps Script web app to a Google Sheet that the administrator owns. We do not host or receive this data; it goes directly to the administrator's own Google account.

4. What we do NOT do

  • We do not sell or transfer user data to third parties, outside of the approved use cases described above.
  • We do not use or transfer user data for purposes that are unrelated to the Extension's single purpose.
  • We do not use or transfer user data to determine creditworthiness or for lending purposes.
  • We do not include any third-party analytics, ad networks, or trackers in the Extension.
  • We do not load remote code at runtime. All scripts are bundled in the published extension package.

5. Permissions and why they are needed

  • tabs — open the part page after a scan, focus the popup window, and close tabs after Add Images or Delete completes.
  • storage, unlimitedStorage — save user preferences and the historical photoLog locally.
  • clipboardWrite — copy the Reference number to the clipboard in Work Order mode.
  • alarms — retry failed Google Sheets sync attempts in the background.
  • Host permissions on app.dismantly.ai, api.dismantly.ai, script.google.com, script.googleusercontent.com — the only origins the Extension communicates with.

6. Data retention and deletion

All locally stored data remains on the user's device until the user uninstalls the Extension or clears its storage from chrome://extensions. Uninstalling the Extension removes the local photoLog and all preferences from that browser profile. If the Google Sheets sync was used, deletion of historical records in the spreadsheet must be performed by the administrator who owns the sheet.

7. Children

The Extension is a workplace productivity tool and is not directed to children under 13.
